# Layer 5 — The Session Layer

**Purpose:**  
The Session Layer controls the **start, maintenance, and end** of communication sessions between the applications.

---

## What Happens at This Layer?
- Establishes, manages, and terminates sessions
- Handles authentication in some instances
- Manages continuous data exchange (e.g., video calls)
- Keeps different conversations (sessions) separate

---

## Real-World Examples
- API session authentication tokens
- Remote procedure calls (RPC)
- NetBIOS
- Some parts of SSL/TLS negotiation

---

## Summary
Layer 5 organizes the exchange of data between systems so applications can communicate easier .
